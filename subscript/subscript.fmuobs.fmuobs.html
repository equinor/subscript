subscript.fmuobs.fmuobs¶

fmuobs is a converter tool for observation files used in assisted history matching

class subscript.fmuobs.fmuobs.CustomFormatter(prog, indent_increment=2, max_help_position=24, width=None)¶

Multiple inheritance used for argparse to get both defaults and raw description formatter

class subscript.fmuobs.fmuobs.FmuObs(ert)¶

This class defines the ERT workflow hook.

It is constructed to work identical to the command line except

  • fmuobs is upper-cased to FMUOBS

  • All option names with double-dash must be enclosed in “” to avoid interference with the ERT comment characters “–”.

run(*args)¶

Pass the ERT workflow arguments on to the same parser as the command line.

subscript.fmuobs.fmuobs.autoparse_file(filename)¶

Detects the observation file format for a given filename. This is done by attempting to parse its content and giving up on exceptions.

NB: In case of ERT file formats, the include statements are interpreted relative to current working directory. Thus it is recommended to reparse with correct cwd after detecting ERT file format. The correct cwd for include-statement is the path of the ERT config file, which is outside the context of fmuobs.

Parameters

filename (str) –

Returns

First element is a string in [resinsight, csv, yaml, ert], second element is a dataframe or a dict (if input was yaml).

Return type

tuple
