subscript.bjobsusers.bjobsusers

subscript.bjobsusers.bjobsusers.call_bjobs(status='RUN')

Call the system bjobs utility

Filter on a specific status, and to only the username and the nodecount

Parameters:

status (str) – A string that the bjobs output will be grepped to should typically be RUN or PEND

Returns:

Multiline string in ascii, looking like:

foobart 4*computenode1
foobarter 2*computenode4
foober computenode1

where the optional number in front a compute node name denotes the number of allocated cores to the job.

Return type:

str

subscript.bjobsusers.bjobsusers.call_finger(username)

Call the system utility ‘finger’ on a specific username

Return type:

str

Returns:

Unicode string with the first line of output from ‘finger’ Example return value:

Login: foobert      Name: Foo Barrer (FOO BAR COM)"

subscript.bjobsusers.bjobsusers.get_jobs(status, bjobs_function)

Make a Pandas dataframe out of the bjobs output

Sums the cpu/core usage pr. user.

Parameters:

  • status (str) – Type of job to list, RUN or PEND

  • bjobs_function (Callable) – Function handle to a function that can return a string with bjobs output.

Returns:

Dataframe with the columns user and ncpu. Only one row pr username. Sorted descending by ncpu.

Return type:

pd.DataFrame
