subscript.csv_stack.csv_stack.csv_stack(dframe, stackmatcher, stackseparator, newcolumn)

Reshape an incoming dataframe by stacking/pivoting.

The dataframe object will be modified in-place.

Parameters

  • dframe (pd.DataFrame) – Data to reshape

  • stackmatcher (Pattern) – Regular expression that matches columns to be stacked.

  • stackseparator (str) – String to use for splitting columns names

  • newcolumn (str) – Name of new column containing the latter part of the stacked column names.

Return type

DataFrame

Returns

pd.DataFrame

subscript.csv_stack.csv_stack.csv_stack_main(args, support_magics=False)

A main function to be used both from the command line, and when used as an ERT plugin (ERT workflow).

This function writes to disk or to stdout.

Parameters

  • args (argparse.Namespace) – Namespace with command line arguments

  • support_magics (bool) – If True, it is possible to read and write to stdin/stdout. Should not be set when used as ERT workflow.

subscript.csv_stack.csv_stack.drop_constants(dframe, keepminimal, stackmatcher)

Drop/purge/remove columns from the dataframe that we don’t want to include in a stacking operation (stacking can blow up the dataframe size)

Obey a global variable with list of columns that we should always keep, you don’t want to remove e.g. the ensemble name even if it is constant.

Parameters

  • dframe (pd.DataFrame) – Dataframe with data

  • keepminimial (bool) – If True, columns not involved in the stacking operation will also be dropped.

  • stackmatcher (Pattern) – Regular expression that matches the columns to be stacked.

Return type

DataFrame

Returns

pd.DataFrame, possibly with fewer columns.
