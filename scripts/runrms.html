RUNRMS¶

Script to run a rms project from command line, which will in turn use the ‘rms…’ command OR will look at /prog/roxar/site. Note that not all options valid for ‘rms’ should be covered.

  • It should understand current RMS version in project and launch correct RMS executable

  • It should be able to run test versions of RMS

  • It should be able to set the correct Equinor valid PYTHONPATH.

  • Company wide plugin path

Example of usage:

runrms newreek.rms10.1.3 (if new project: warn and just start rms default)
runrms reek.rms10.1.3  (automatically detect version from .master)
runrms -project reek.10.1.3  (same as previous)
runrms reek.rms10.1.3 -v 11.0.1 (force version 11.0.1)


usage: runrms [-h] [--debug] [--dryrun] [--version RVERSION] [--beta]
              [--readonly] [--dpiscaling SDPI]
              [--batch BWORKFLOWS [BWORKFLOWS ...]] [--nopy] [--includesyspy]
              [--testpylib] [--setup ALTSETUP]
              [project]

Positional Arguments¶

project

RMS project name

Named Arguments¶

--debug

If you want to run this script in verbose mode

Default: False

--dryrun

Run this script without actually launching RMS

Default: False

--version, -v

RMS version, e.g. 10.1.3

--beta

Will try latest RMS (alpha, beta or test) version, alternative -v latest

Default: False

--readonly, -r, -readonly

Read only mode (disable save)

Default: False

--dpiscaling, -d

Specify RMS DPI display scaling as percent, where 100 is no scaling

Default: 100

--batch, -batch

Runs project in batch mode (req. project) with workflows as argument(s)

--nopy

If you want to run RMS withouth any modication of current PYTHONPATH

Default: False

--includesyspy

If you want to run RMS and include current system PYTHONPATH (typically Komodo based)

Default: False

--testpylib

If you want to run RMS and use a test version of the Equinor installed Python modules for RMS, e.g. test XTGeo (NB special usage!)

Default: False

--setup

Alternative path to runrms.yml (NB special usage/testing!)
