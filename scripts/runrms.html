RUNRMS

Script to run a rms project from command line, which will in turn use the ‘rms…’ command OR will look at /prog/roxar/site. Note that not all options valid for the base script ‘rms’ should be covered.

  • It should understand current RMS version from project and launch correct RMS executable

  • It should be able to run test versions of RMS

  • Set the correct Equinor valid PYTHONPATH.

  • Set company wide plugin path

Example of usage:

  • runrms newreek.rms10.1.3 (if new project: warn and just start rms default)

  • runrms reek.rms10.1.3 (automatically detect version from .master)

  • runrms -project reek.10.1.3 (alternative to previous, but using -project as in former rms command)

  • runrms reek.rms10.1.3 -v 11.0.1 (force version 11.0.1)

Notes:

  1. The ‘runrms’ will be aliased to ‘rms’ in Equinor after summer 2022. When that implemented, running the original ‘rms’ script can still be done with (where the ‘$’ means terminal prompt):

    $ /prog/roxar/rms/rms.

  2. For backward compatibility a project name can be prepended with -project. Hence, to specify a project use either: $ runrms myproject.rms13.0.3 which is preferred, or

    $ runrms -project myproject.rms13.0.3

    or

    $ runrms –project myproject.rms13.0.3

    If both are used, e.g.:

    $ runrms foo.rms13.0.3 -project bar.rms12.1.2

    then the alternative without -project will ‘win’, here ‘foo.rms13.0.3’ and a warning will be issued.

usage: runrms [-h] [--project PROJECT_ALT] [--listversions] [--debug]
              [--dryrun] [--version [RVERSION]] [--beta] [--readonly]
              [--dpiscaling SDPI] [--batch BWORKFLOWS [BWORKFLOWS ...]]
              [--seed SEED] [--nopy] [--includesyspy] [--testpylib]
              [--setup ALTSETUP]
              [project]

Positional Arguments

project

RMS project name

Named Arguments

--project, -project

RMS project name. Alternative option for backward user experience compatibility. See details in Note (2) above

--listversions, -l

Use this option to list current RMS versions available. If this option is set then all other options are disabled

Default: False

--debug

If you want to run this script in verbose mode

Default: False

--dryrun

Run this script without actually launching RMS

Default: False

--version, -v

RMS version, e.g. 10.1.3

--beta

Will try latest RMS (alpha, beta or test) version, alternative -v latest

Default: False

--readonly, -r, -readonly

Read only mode (disable save)

Default: False

--dpiscaling, -d

Specify RMS DPI display scaling as percent, where 100 is no scaling

Default: 100

--batch, -batch

Runs project in batch mode (req. project) with workflows as argument(s)

--seed, -seed

Runs project with seed number set. Needs to be combined with –batch and project!

--nopy

If you want to run RMS withouth any modication of current PYTHONPATH

Default: False

--includesyspy

If you want to run RMS and include current system PYTHONPATH (typically Komodo based)

Default: False

--testpylib

If you want to run RMS and use a test version of the Equinor installed Python modules for RMS, e.g. test XTGeo (NB special usage!)

Default: False

--setup

Alternative path to runrms.yml (NB special usage/testing!)
